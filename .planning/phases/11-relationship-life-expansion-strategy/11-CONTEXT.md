# Phase 11 Context

Phase: 11
Name: Relationship Life Expansion Strategy
Date: 2026-03-31

## Source Of Truth

- `SPARQ_MASTER_SPEC.md`

## Why This Phase Exists

Sparq now has a credible serious-core path:
- strong personalization
- emotionally intelligent guidance
- a proven signup-driven primary path

What it does not yet have is enough lightness, fun, and everyday pull to become a relationship life app instead of only a relationship-work app.

This phase exists to define the next product layer without redesigning the proven path or inventing engineering work too early.

## Goal

Expand Sparq’s product direction from:
- repair
- growth

into:
- repair
- growth
- enjoyment, play, and shared connection

## Constraints

In scope:
- define a product map for lighter, more joyful relationship use cases
- rank the opportunity areas
- explain user value, retention value, fit with Sparq’s identity, and rollout timing

Out of scope:
- engineering plans
- implementation tasks
- code changes
- redesigning the proven primary path

## Output Needed

A structured product map with these buckets:
- playful connection layer
- shared rituals layer
- adaptive date-night layer
- couple game mode
- joy and celebration layer

For each bucket:
- user value
- why it increases retention
- how it fits Sparq’s identity
- examples of features
- implementation complexity
- beta risk
- whether it should be built now, later, or much later

And a final ranked roadmap.
