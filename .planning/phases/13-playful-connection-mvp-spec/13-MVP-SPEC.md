# Phase 13 MVP Spec

Date: 2026-03-31
Phase: Playful Connection MVP Spec

## MVP Shape

This MVP should add a light layer beside Sparq’s serious core, not replace it.

Recommended feature count:
- 4 features

Why 4:
- enough variety to make Sparq feel alive
- still small enough to build in a contained slice
- covers connection, delight, laughter, and enjoyment without expanding into a full game system

## Feature 1: Daily Spark

Category:
- connect us

### User story

As a user, I want one small playful prompt each day so I can feel closer without turning the moment into heavy relationship work.

### Why it matters

- gives users a reason to open Sparq on normal days
- creates a lightweight habit loop
- keeps connection active before problems build

### Entry point in the app

- dashboard
- optional companion placement on daily-growth home

### Frequency of use

- once per day

### Required backend or data needs

- prompt library with tags such as:
  - solo
  - partner_optional
  - tone
  - energy
  - prompt_type
- per-user state:
  - shown
  - swapped
  - tried
  - sent
- basic analytics for interaction and next-day return

### UX flow

1. user sees one short prompt
2. user can choose:
   - try it
   - send it
   - swap it
3. optional one-line reflection after using it

### Beta scope

- one prompt per day
- curated prompt library only
- no history
- no packs
- no scoring

### Future expansion path

- favorites
- spark history
- better personalization by mood and journey
- partner reactions

## Feature 2: Favorite Us

Category:
- help us enjoy each other

### User story

As a user, I want a simple prompt that helps me notice what I enjoy about us so the relationship feels more alive and not just something I work on when it is hard.

### Why it matters

- shifts attention toward what is good
- deepens positive memory and enjoyment
- keeps Sparq emotionally intelligent instead of becoming random fun content

### Entry point in the app

- dashboard card rotation
- optional evening add-on after the core daily loop

### Frequency of use

- 2 to 3 times per week

### Required backend or data needs

- prompt library tagged:
  - appreciation
  - memory
  - ease
  - enjoyment
- optional saved response field for favorite moments
- lightweight analytics for prompt usage and save rate

### UX flow

1. user receives a short “Favorite Us” prompt
2. user can answer privately or send a short version to partner
3. optional save as a favorite moment

### Beta scope

- prompt plus optional save
- no archive view yet
- no timeline yet

### Future expansion path

- joy archive
- anniversary recap
- “best of us” collections
- celebration layer integration

## Feature 3: Tiny Dare

Category:
- delight us

### User story

As a user, I want a tiny challenge that helps us surprise each other or break routine in a way that feels warm and easy.

### Why it matters

- creates novelty without a big time ask
- turns Sparq into action, not just reflection
- gives couples something to do on good days

### Entry point in the app

- dashboard rotation
- optional “need something lighter?” surface

### Frequency of use

- 2 to 3 times per week

### Required backend or data needs

- dare library tagged:
  - energy
  - public_private
  - solo
  - partner_optional
- completion and skip state
- simple analytics for try rate and completion rate

### UX flow

1. user sees one tiny dare
2. user chooses:
   - do it
   - skip
   - save for later
3. optional “done” moment after action

### Beta scope

- 10 to 15 curated dares
- no streak system
- no unlock structure

### Future expansion path

- weekend dares
- surprise mode
- partner-coordinated dares
- delight streaks

## Feature 4: Laugh Loop

Category:
- make us laugh

### User story

As a user, I want playful prompts that help us laugh together so the app feels alive and the relationship feels lighter.

### Why it matters

- laughter is one of the fastest ways to create delight
- reduces the sense that Sparq is only serious or clinical
- adds humanity without leaving the brand’s emotional depth

### Entry point in the app

- rotating playful card on dashboard
- optional lighter-mode entry from daily-growth home

### Frequency of use

- 2 to 4 times per week

### Required backend or data needs

- humor-safe prompt library tagged:
  - laugh
  - memory
  - inside_joke_potential
  - solo
  - partner_optional
- simple usage analytics

### UX flow

1. user receives a laugh-oriented prompt
2. user can use it live, send it, or swap it
3. optional “that landed” tap if it created a good moment

### Beta scope

- prompt-only
- no live multiplayer
- no inside-joke memory engine yet

### Future expansion path

- couple-specific humor memory
- funny recap cards
- inside-joke collections
- celebration tie-ins

## Recommended MVP Order

1. Daily Spark
2. Favorite Us
3. Tiny Dare
4. Laugh Loop

## Why This Set Works

- Daily Spark creates the clearest daily entry habit.
- Favorite Us protects Sparq’s emotional depth.
- Tiny Dare adds motion and surprise.
- Laugh Loop adds warmth and humanity.

Together they make Sparq feel lighter without turning it into a game app.

## Surfaces Across The MVP

Primary surfaces:
- dashboard
- daily-growth home as an optional companion surface

Secondary surfaces:
- light reminder or notification surface
- simple send or share flow

Not in beta MVP:
- new standalone playful tab
- new game hub
- deep history or collections
- complex social loops

## Beta Boundaries

- all playful features stay optional
- nothing blocks the proven signup-driven path
- nothing replaces the core daily-growth loop
- no packs, points, levels, or heavy progression systems yet
- prompts must stay emotionally aware and simple

## Recommendation

If Sparq wants a contained playful MVP with real value, it should launch these four features as one light product layer around the existing core:

1. Daily Spark
2. Favorite Us
3. Tiny Dare
4. Laugh Loop

If even that feels too large, the smallest first slice is:

- Daily Spark
- Favorite Us

That smallest pair is enough to make Sparq feel more alive without widening scope too fast.
