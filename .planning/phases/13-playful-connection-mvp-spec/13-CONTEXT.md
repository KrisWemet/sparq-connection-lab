## Why This Phase Exists

Phase 12 selected the playful connection layer as Sparq’s best next expansion lane and defined the smallest valuable MVP shape.

The next product step is to turn that into a feature-level MVP spec that is still:
- contained
- emotionally intelligent
- safe for the proven primary path

without drifting into engineering planning.

## Goal

Define a playful connection MVP made of 3 to 5 features that make Sparq feel:
- lighter
- more fun
- more alive

while staying native to Sparq and useful on good days, not just hard ones.

## Constraints

In scope:
- 3 to 5 playful features only
- user-facing product definition
- app entry points
- usage cadence
- high-level backend or data needs
- UX flow
- beta scope
- future expansion path

Out of scope:
- code
- engineering tasks
- architecture design
- redesign of the proven signup-driven path

## Required Feature Categories

Each feature must fit one of:
- connect us
- delight us
- make us laugh
- help us enjoy each other
