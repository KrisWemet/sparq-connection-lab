## Why This Phase Exists

Sparq's product direction has moved beyond "serious relationship work only."

The app now needs a visual system that feels:
- more editorial
- more premium
- more alive
- more browseable on good days
- more aligned with a relationship life app

without damaging what is already proven:
- signup-driven onboarding
- dashboard arrival
- daily-growth start
- Day 1 completion

## Goal

Turn the new aesthetic target into a concrete UI contract for safe existing surfaces so a later implementation phase can refresh the product without guessing.

## Safe Surfaces

In scope:
- dashboard shell and cards
- daily-growth home shell and companion modules
- bottom navigation treatment
- editorial content cards and supporting modules
- playful MVP card styling on current surfaces

Out of scope:
- onboarding logic changes
- Peter behavior changes
- signup and auth changes
- daily-growth step logic changes
- new tabs, new sections, or new product features

## Success Shape

The contract should make Sparq feel:
- warm
- elegant
- lightweight
- emotionally safe
- couple-friendly
- premium but not intimidating

while still staying:
- simple to read
- low friction
- additive to the proven primary path
