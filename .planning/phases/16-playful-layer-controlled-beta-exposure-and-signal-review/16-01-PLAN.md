# 16-01 Plan

Phase: 16
Wave: 1
Title: Controlled-Beta Exposure And Cohort Signal Capture

## Objective

Expose the existing playful MVP only to the current controlled-beta cohort and capture the core funnel plus playful interaction signals needed for a real usage readout.

## Scope

- expose the playful slice only to the current controlled-beta cohort
- keep the slice limited to its existing live surfaces:
  - `Daily Spark` on dashboard
  - `Favorite Us` on the daily-growth home
- monitor the core funnel and playful interaction stream together
- confirm the playful slice remains optional and non-blocking during controlled exposure

## Tasks

1. verify that the playful MVP is only exposed to the intended controlled-beta cohort
2. define the exact cohort observation window and evidence window for this phase
3. capture the core funnel events for users in the observation window:
   - `beta_primary_signup_register_success`
   - `beta_primary_journey_selected`
   - `beta_primary_onboarding_completed`
   - `beta_primary_dashboard_arrived`
   - `beta_primary_daily_growth_started`
   - `beta_primary_day1_completed`
4. capture the playful interaction events for the same observation window:
   - `playful_daily_spark_viewed`
   - `playful_daily_spark_tried`
   - `playful_daily_spark_swapped`
   - `playful_daily_spark_sent`
   - `playful_favorite_us_viewed`
   - `playful_favorite_us_saved`
   - `playful_favorite_us_sent`
5. record whether the primary signup-driven path remains stable while the playful slice is exposed
6. preserve exact cohort evidence in a new Phase 16 evidence file

## Acceptance Criteria

- the playful slice is exposed only to the intended controlled-beta cohort
- the core funnel remains queryable during the observation window
- the playful interaction stream remains queryable during the same window
- there is enough real cohort data to support a later decision memo
- no regression to the proven primary path is uncovered during the exposure window

## Verification

- production `analytics_events` queries scoped to the controlled-beta cohort and observation window
- targeted production spot-checks of:
  - dashboard load
  - daily-growth load
  - presence of the current playful surfaces for exposed users
- operator notes that tie the observed cohort to the production evidence window

## Notes

- do not widen exposure beyond the current controlled-beta cohort
- do not change the current playful feature set or placement in this plan
