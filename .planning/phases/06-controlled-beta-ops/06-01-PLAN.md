# Plan 06-01 — Beta Feedback Capture

Wave: 1

## Objective

Give beta users on the primary signup-driven path a simple way to submit feedback without changing the core product experience.

## Scope

- add a lightweight feedback capture path tied to the proven primary journey
- keep the feedback surface small, optional, and clearly beta-oriented
- ensure captured feedback is attributable to the stage of the proven funnel where it was submitted

## Expected Changes

- feedback data model or API path
- small feedback entry point on the proven primary path
- basic operator visibility into submitted beta feedback

## Explicitly Out Of Scope

- new community or messaging features
- broad in-app support tooling
- redesign of onboarding, dashboard, or daily-growth

## Verification

- `npm run lint`
- focused tests for feedback submission on the primary path
- manual beta-path verification that feedback can be submitted without breaking the flow

## Success Criteria

- a beta user can submit feedback from the primary path
- feedback is stored with enough context to be actionable
- the feedback entry point does not interrupt the proven Day 1 flow
