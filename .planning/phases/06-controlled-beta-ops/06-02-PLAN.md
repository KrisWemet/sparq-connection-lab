# Plan 06-02 — Primary Path Funnel Analytics

Wave: 1

## Objective

Instrument the proven primary signup-driven path so controlled beta operators can measure where users progress and where they drop.

## Scope

- instrument only the primary path events:
  - signup/register success
  - onboarding completion
  - journey selection
  - dashboard arrival
  - daily-growth start
  - Day 1 completion
- make event names and payloads consistent and easy to query
- avoid expanding instrumentation to non-beta or secondary flows unless needed for context

## Expected Changes

- analytics event naming cleanup for the primary path
- event emission at the key funnel milestones
- documentation of the funnel event contract

## Explicitly Out Of Scope

- full product analytics expansion
- partner-path analytics
- retention dashboards beyond the proven primary funnel

## Verification

- `npm run lint`
- focused tests asserting funnel events fire on the primary path
- manual verification of event writes for a sample user journey

## Success Criteria

- the full proven primary path is measurable stage by stage
- event names are stable enough for beta reporting
- operators can distinguish path success from drop-off
