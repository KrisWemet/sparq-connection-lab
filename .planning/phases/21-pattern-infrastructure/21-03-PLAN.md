2. Replace the direct `profile_traits` query in the `Promise.all` block with `buildPatternContext`.

**For morning.ts (lines 91-96 inside the Promise.all):**

Replace:
```typescript
authed.supabase
  .from('profile_traits')
  .select('trait_key, inferred_value, confidence, effective_weight')
  .eq('user_id', authed.userId)
  .gte('effective_weight', 0.3),
```

With:
```typescript
buildPatternContext(authed.supabase, authed.userId),
```

Then update the destructuring and trait construction. The `Promise.all` result for this slot changes from `traitsResult` (a Supabase query result with `.data`) to a `PatternContext` object directly.

Update the line that builds `traits`:
```typescript
// BEFORE:
const traits: ProfileTrait[] = traitsResult.data || [];

// AFTER:
const traits: ProfileTrait[] = patternContextToTraits(patternContext);
```

Where `patternContext` is the name used in the destructured Promise.all result.

The rest of the code (`buildPersonalizedPrompt(traits, memories, ...)`) stays exactly the same — it receives a `ProfileTrait[]` as before.

**For chat.ts (lines 102-107 inside the Promise.all):**

Same transformation — replace the direct query with `buildPatternContext`, destructure as `patternContext`, convert via `patternContextToTraits`.

**Important behavioral notes:**
- The existing `try/catch` around the personalization block stays in place — `buildPatternContext` also never throws, but the outer catch protects against other personalization failures
- `love_language` and `conflict_style` traits will no longer be included in the `ProfileTrait[]` passed to `buildPersonalizedPrompt` because `patternContextToTraits` only returns the 8 pattern dimensions. However, since Phase 21's `TRAIT_DESCRIPTIONS` only has entries for `attachment_style`, `love_language`, and `conflict_style`, and `buildPersonalizedPrompt` silently skips traits without descriptions, the only net effect is that `love_language` and `conflict_style` personalization lines will be absent. **To preserve full backward compatibility:** in the `Promise.all`, keep a separate query for the non-pattern traits (`love_language`, `conflict_style`) and merge the results. OR: change the `buildPatternContext` call to also return the full trait array alongside the PatternContext.

**Recommended approach for backward compatibility:** Replace the direct query with `buildPatternContext` for the pattern context, but also query the remaining traits (`love_language`, `conflict_style`) separately. Then merge:

```typescript
const [patternContext, remainingTraitsResult, profileResult, insightsResult, memResult] = await Promise.all([
  buildPatternContext(authed.supabase, authed.userId),
  authed.supabase
    .from('profile_traits')
    .select('trait_key, inferred_value, confidence, effective_weight')
    .eq('user_id', authed.userId)
    .in('trait_key', ['love_language', 'conflict_style'])
    .gte('effective_weight', 0.3),
  // ... profileResult, insightsResult, memResult queries stay unchanged
]);

const traits: ProfileTrait[] = [
  ...patternContextToTraits(patternContext),
  ...(remainingTraitsResult.data || []),
];
```

This satisfies D-18 (removes the independent query for the 8 pattern keys) while preserving `love_language` and `conflict_style` personalization.
