**Section 2 — attachment_style data migration (per D-11):**

```sql
UPDATE profile_traits
SET
  inferred_value = CASE inferred_value
    WHEN 'anxious'      THEN 'reaches_out'
    WHEN 'avoidant'     THEN 'steps_back'
    WHEN 'disorganized' THEN 'feels_torn'
    WHEN 'secure'       THEN 'feels_steady'
    ELSE inferred_value
  END,
  updated_at = now()
WHERE trait_key = 'attachment_style'
  AND inferred_value IN ('anxious', 'avoidant', 'disorganized', 'secure');
```

Use CASE expression (not DELETE+INSERT) to preserve `confidence`, `effective_weight`, `user_feedback`, and `updated_at` on existing rows.

No DDL changes (no new tables, no new columns) — the 7 new keys are just new row values in the existing `profile_traits` table.
