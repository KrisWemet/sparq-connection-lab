# Plan 05-01 — Deterministic Onboarding Handoff

Wave: 1

## Objective

Make the live Peter onboarding phase deterministic enough for a fresh production user to reliably reach journey selection without getting stuck in an open-ended conversation branch.

## Scope

- inspect the live Peter onboarding close conditions against the current onboarding state machine
- tighten the close/handoff logic only where it blocks a deterministic transition into `journey_rec`
- keep solo-first behavior and current product framing intact
- add or update focused automated coverage for the deterministic handoff behavior

## Expected Changes

- onboarding conversation close criteria
- handoff timing or state synchronization between `PeterSession` and `/onboarding`
- focused Playwright or integration coverage for the handoff branch

## Explicitly Out Of Scope

- redesigning Peter’s conversation experience
- expanding onboarding questions
- changing the overall onboarding architecture

## Verification

- `npm run lint`
- targeted onboarding tests
- targeted live-like or mocked coverage that proves the handoff reaches journey selection reliably

## Success Criteria

- a fresh user can reliably move from Peter onboarding into visible journey selection
- the handoff no longer depends on variable timing or unbounded conversation length
- recommendation selection is deterministic enough to support the live beta proof run
