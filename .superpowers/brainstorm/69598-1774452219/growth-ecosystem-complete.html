Journey Completion Flow

STEP 1: COMPLETION
Peter delivers journey synthesis — what shifted from Day 1 to now. Growth Thread gets a milestone entry.
STEP 2: USER DECIDES
Peter asks: "Ready to keep going? Or would you like a day to let this settle?" User has full agency.
STEP 3: NEXT JOURNEY
Peter recommends 3 based on what emerged (not just original profile). User can also browse all unlocked.
STEP 4: DASHBOARD UPDATES
Peter's greeting reflects completion. Journey Arc shows finished arc. Practice CTA shifts to "Choose Next Journey."

Data Architecture (New)

Evening check-in data: practice_attempted, reflection_text, peter_observation, emotional_tone
Growth thread entries: date, label, type (milestone/breakthrough/pattern/mirror/pinned), journey_id, detail
Weekly mirror: week_start, narrative_text, practice_count, practice_natural_count, key_patterns
Peter greeting: pre-generated at session end, stored in daily_sessions or user_insights
Journey completion: synthesis_text, day1_vs_now comparison, recommended_next_journeys[]

✓ All Decisions Captured

  • Three growth loops: Practice (daily) → Pattern (weekly) → Story (per journey)
  • Living Dashboard: Peter's greeting + modular cards blend
  • Peter's greeting: pre-generated at session end
  • Evening check-in: adaptive (short or deep based on user response)
  • Growth Thread: AI-curated + user-pinnable moments
  • Weekly Mirror: generated Sunday evening
  • Journey completion: Peter synthesizes, user decides rest vs. continue
  • Next journey: Peter recommends 3 based on journey outcomes
  • Solo-first, partner-ready architecture
