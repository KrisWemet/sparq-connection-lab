API design — /api/peter/onboarding
// POST /api/peter/onboarding
// Request body:
{
  "messages": PeterMessage[], // full conversation so far
  "profile": DerivedProfile, // from deriveProfile()
  "exchangeCount": number // 1–5, Peter closes at 5
}

// Response:
{
  "message": string,
  "shouldClose": boolean, // Peter decided he has enough
  "safety": { triggered: boolean }
}

// No entitlement checks. No usage caps.
// Crisis detection: YES (always on).
// System prompt built from profile.attachmentStyle
// + profile.toneMode + freeTextAnswers.
Peter's closing signal: The system prompt instructs Peter to include READY_TO_CLOSE as a hidden marker in his response JSON when he has enough. The client strips this before rendering — it just triggers the transition to journey recommendation.
